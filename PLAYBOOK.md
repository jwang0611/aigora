# AIgora Playbook — 实测校准的创作流程建议（短篇 / 长篇）

> **地位**：建议层（advisory）。门控与管线的权威定义不变，仍在
> `references/core/quality-rubric.md`、`references/core/longform-quality-gates.md`、
> `AUTOPILOT.md`、`LONGFORM.md` —— 本文与规格冲突处，以规格为准。
> 标注【规格修订建议】的条目尚未入规格，采纳需另行 PR（汇总见 §6）。
>
> **依据**：三次端到端实测 ——《慢房间》(PR #2，短篇 Autopilot)、《一比一》(PR #4，
> LONGFORM 试点) 、《三缄》(PR #5，短篇 Autopilot，首次触发 ESCALATE) —— 以及
> 《一比一》复盘催生的规格修订 (PR #6)。本文的每条断言都能在这三个项目目录的
> `scores.md` / `decisions.md` / `critiques/` 里找到出处。

---

## 0. 证据基础：三次实测一览

| | 《慢房间》 | 《一比一》 | 《三缄》 |
|---|---|---|---|
| 管线 | Autopilot 短篇 | LONGFORM 试点 | Autopilot 短篇 |
| 体量 | ~6,000字 / 5节 | 46,276字 / 2卷12章 | ~8,400字 / 5节 |
| 前提类型 | 规则驱动（门内十年） | 规则驱动（1:1 数字孪生） | 类型正面强攻（武侠三大件） |
| 人工介入 | 仅 brief | 仅 brief | 仅 brief |
| 评分轨迹 | 51→51（终验53） | 卷 65/80、68/80；书 50/60 | 43→42→45→45 |
| 结果 | PASS（2轮） | 全门 PASS r1 | **ESCALATE**（未过线） |
| 盲评人次 | 7 | 12 | 12 |

**五条经过验证的事实**：

1. **框架可靠保下限，从不保上限。** 连续性、去AI味、伏笔闭合、结构因果，三次全部
   兑现；但 Originality 与修订轮数无关，与前提新鲜度完全同步。
2. **上限在构思期锁定。**《三缄》三轮修订修掉了全部结构性缺陷（火灾因果两次重做、
   时间线、笔迹证据链、视角越界），Structure/Character 升到 8，Originality 纹丝不动
   （6→7 徘徊）。**修订买得回缺陷，买不回新意** —— 这个结论花了 12 人次盲评的代价。
3. **盲评的不可替代价值 = 看见起草上下文看不见的东西。**《一比一》主上下文审计只抓到
   年份不统一；全新上下文的盲评抓出坡顶高程矛盾、迁坟时间线冲突、水位倒退。
4. **评审会自信地报错、互相矛盾。** 两个盲评对同一农历日期给出相反的"更正"，照单
   全收会把对的改成错的 —— 主编对真值的裁决不可省略（已入规格：Must Fix 裁决制）。
5. **Rubric 驱动会把作品推向过度闭合。**《一比一》书级评语："不像没写完，像打磨过头"。
   反过度闭合要在**大纲期**注入（章末振幅变奏、open-by-design 留白），评后补救无效
   （封卷不重开）。

## 1. 三条经济学法则（决定哪里省、哪里不省）

**法则一：人的时间买天花板，机器的时间买下限。**
人工 15 分钟花在 brief 和内核确认上，价值超过任何数量的评后打磨。《三缄》整个
4 轮循环（12 人次盲评）本可被一次内核层面的人工换向避免（推断，非实证——但两个
过线项目与一个卡线项目的唯一系统性差异就是前提新鲜度）。

**法则二：修订的边际收益在第 2 轮后急剧递减。**
实测总分变化：《慢房间》r1→r2 +0（但 Must Fix 全闭合）；《三缄》−1 / +3 / +0。
修订的真实产出是**缺陷闭合**，不是提分。规格的 3 轮上限是天花板，不是配额 ——
用满它通常不是好信号。

**法则三：机械审计便宜且必须，品味评审昂贵且可调。**
ledger 核对、recap 准确性、cap 测量（`wc -m`）、burn-rate 都是 grep 级成本，任何
模式下都不省；**盲评人次是主要开销**（约等于起草成本再翻一倍），是唯一值得按风险
调节的旋钮。

## 2. 短篇建议流程（< ~3万字，Autopilot 管线）

### 2.1 各阶段：谁做、多重要、能不能省

| 阶段 | 建议执行 | 重要性 | 放宽空间 |
|---|---|---|---|
| **S0 Brief** | **人写，10 分钟**（清单见 2.2） | ★★★ 决定分数段 | 【不可省】"你来定"合法，但等于把天花板交给一个称职的均值 |
| **S1 构思 + 内核确认** | 机器讨论自决；**内核由人确认（5 分钟）** | ★★★ 天花板锁定点 | 省人模式可跳过，接受《三缄》型卡线风险 |
| **S2 大纲** | 全自动 + **大纲期注入反闭合**（章末落法变奏、1 处 open-by-design） | ★★ | 人工确认大纲【可省】——三次实测 EIC 的大纲决策无一失手 |
| **S3 起草** | 全自动，**串行**，逐章 ledger + L0 检查 | ★★★ 纪律成本≈0 | 【不可省】短篇永远串行（规格） |
| **S4 盲评** | 3 评审并行全新上下文 × 中位 | ★★★ 全流程最硬的一环 | 人次可调（2.3）；**"全新上下文"属性【不可省】** |
| **S5 修订** | EIC 自决，建议默认 2 轮 | ★★ | 提前退出规则（2.4） |

### 2.2 Brief 怎么写（10 分钟清单，最高杠杆的 10 分钟）

1. **一个具体到发疼的张力**，不是主题。"最后一个方言使用者的临终病房"是故事，
   "关于记忆的故事"什么都不是（主编决策原则原文如此）。
2. **前提新鲜度自检**（《三缄》教训）：同类型老读者能否从第一句话预测你的前提？
   能 → Originality 封顶 6–7 → 过线（48）要求其余五维平均 8.2，极难。
   **规则驱动前提**（一条设定规则产生全部剧情推论）是两次实测验证的甜区：
   《慢房间》三条门内规则、《一比一》"1:1 副本装不进一个名字"。
3. **复合情感目标**（悲伤里有解脱、爱里有怨）—— 量表把单一情绪封顶在 7。
4. **一行保留决策**，把"平衡模式"装进现行规格，无需任何规格修改：
   `Reserved decisions: 故事内核与结局方向须经我确认`。

### 2.3 盲评人次的调节

- **推荐默认：3 评审取中位。** 两次短篇实测中 R1（素读者）与 R3（结构）总分相差
  ≤3；R2（敌意）是分歧来源（《慢房间》低于同轮均值约 5 分、《三缄》约 8 分）。
  三人中位对单个离群评审稳健 —— 这是设计目的，实测成立。
- **省人变体：2 评审（R2 敌意 + R3 结构）**【规格修订建议】：适用于**人已确认内核**
  的稿件。Must Fix 密度最高的两个镜片保留；代价是丢失素读者体验信号 —— 已知一例
  R1 独家发现（《三缄》第 4 轮"两次爆炸"连贯性硬伤是 R1 抓的）。两人分歧时按低分
  记、不取均值，触线即升级。
- **纪律（任何模式不可放宽）**：评审包分发前**冻结修订**。《三缄》第 2 轮评了陈旧
  快照，整轮作废（43→42 的假下跌）【规格修订建议：写入盲评协议】。

### 2.4 修订的提前退出

- 任一维度 < 7 → 修。这是修订真正擅长的（缺陷闭合）。
- **连续两轮只败总分且 Δ总分 ≤ 1 → 立即升级，不烧第 3 轮**。《三缄》r3→r4 用
  3 人次盲评换了 +0 分【规格修订建议】。
- 升级呈报必须给**三个**选项：定稿 / 打磨 / **从大纲重构**。只败总分时，重构（换
  前提角度）是唯一可能突破天花板的选项 ——《三缄》的呈报恰恰漏了它（规格本有要求）。
- 只败总分且总分 ≥ 72% 时：长篇有主编验收阀门，短篇没有。《三缄》45/60=75% 若在
  长篇就能验收。建议补齐，或明文声明"短篇升级便宜、人工裁决合理"的有意不对称
  【规格修订建议】。

### 2.5 短篇三档预设

| | 省人模式 | **平衡模式（推荐）** | 精品模式 |
|---|---|---|---|
| 人工 | brief 1 行 | brief 10min + 内核确认 5min | + 大纲确认 + 终稿亲读 |
| 盲评 | 2 评审 × 1 轮 + 机械复核 | 3 评审 × ≤2 轮 | 3 评审 × ≤3 轮 + 全新评审终验轮 |
| 预期区间* | 45–50，有卡线风险 | 48–53 | 50–55 |
| 适用 | 走量、试方向、选题海选 | 默认 | 投稿、出版 |

\* 基于 3 个数据点的外推，仅供定位，不是承诺。

**启动咒语（平衡模式，可直接粘贴）**：

```
用 AIgora Autopilot 写一篇短篇小说。
Brief：<你的张力/前提，按 PLAYBOOK §2.2 清单写>
Track：文学向
Reserved decisions: 故事内核与结局方向须经我确认（其余全自动）。
盲评用并行子代理，项目目录 projects/<slug>/。
```

## 3. 长篇建议流程（> ~5万字，LONGFORM 管线）

长篇的加速哲学与短篇相反：**短篇省在评审人次，长篇省在人的注意力分配。**
长篇的纪律条目一条都不能省 —— 因为每一条背后都有一个实测事故。

### 3.1 纪律清单（每条都有事故背书，任何模式【不可省】）

| 纪律 | 省掉它的实测后果（《一比一》） |
|---|---|
| 串行起草默认；并行必须四件套（节拍契约 / 清单差量 / 有序账本合并 / 接缝审计） | 水位 322→302 倒退、同一次离村两种讲法、章末悬念被下一章无视 |
| 逐章 ledger + recap 更新；封卷压缩 + `wc -m` 阻断式 cap 审计 | digest 曾超标 2×，三卷后会炸掉加载预算 |
| 回溯修订四联更新（账本 Amendments / digest / 实体 / **reader-recap**） | 陈旧 recap 让下游盲评报出假矛盾，污染其后每一轮评审 |
| Must Fix 主编裁决制（采纳或驳回留证），永不 auto-apply | 两个评审对同一日期给出**相反**的"更正" |
| 封卷后不为品味重开冷卷 | 防"越改越顺、越顺越没声音"的劣化 |
| burn-rate 每卷边界检查 | 规模靠增编故事弧达成，不靠注水 |

### 3.2 人工注意力：只压两个点

1. **Brief + L1 卷地图确认（15 分钟，一次性）**：锁内核、卷地图、书脊 PROMISE。
   实现方式还是保留决策行：
   `Reserved decisions: L1 内核与卷地图须经我确认；卷一封卷前我要亲读`。
2. **卷一亲读（30–60 分钟，全书最后一个便宜的纠偏点）**：卷一是模具 ——
   style-anchor 取自卷一，后续所有卷参照它起草与评审；卷一的缺陷会被复制 12–13 次。
   读三样：卷一正文、`decisions.md` 的 Overruled 行、卷一 `scores.md` 里 R2 的报告。
3. **此后每卷 5 分钟**：只读 `progress.md` 的 Gate history 行 + `decisions.md` 新增段。
   不需要读正文（想读当然可以，但管线不依赖它）。

### 3.3 可放宽点（长篇）

- **卷 2+ 盲评 3→2**【规格修订建议】：连续两卷 PASS r1 且 R1/R3 中位差 ≤1 后，
  文学轨可降为 R2 + R3。**网文轨永远保留 R1** —— 追更读者视角（"晚上 11 点会不会点
  下一章"）是该轨的命，没有代餐。
- **PASS-pending-fixes 用满**（已入规格）：品味过线、只剩客观 Must Fix 时，裁决 +
  定向修复 + 机械复核即封卷 —— 不重评、不耗修订轮次。这是长篇最大的合法省钱项。
- **黄金三章门（网文轨卷一）不可放宽** —— 商业生死线，规格设为 Must Fix 级是对的。
- 修订轮上限维持规格（卷一 3 / 卷 2+ 2），不建议再收紧：卷一是门面，值它的预算。

**启动咒语（长篇推荐配置）**：

```
用 AIgora Longform 写一部长篇小说。
Brief：<前提，按 PLAYBOOK §2.2 清单写>；Track：网文向 或 文学向；
目标规模：<N>万字，<K>卷。
Reserved decisions: L1 内核与卷地图须经我确认；卷一封卷前我要亲读。
卷级盲评用并行子代理，项目目录 projects/<slug>/。
```

中断后恢复：`继续写 projects/<slug>` —— 管线读 `progress.md`，从记录的下一个动作
接着跑。

## 4. 人工时间性价比总表（两种体裁通用）

| 排名 | 介入点 | 花费 | 买到什么 |
|---|---|---|---|
| 1 | Brief（具体张力 + 新鲜前提自检） | 10–15 min | 分数段本身（45 档 vs 52 档） |
| 2 | 内核确认（短篇）/ 卷地图确认（长篇） | 5–15 min | 天花板锁定前唯一的否决权 |
| 3 | 长篇卷一亲读 | 30–60 min | 缺陷不被复制 13 次 |
| 4 | 升级裁决 | 10 min | 只在管线自己停下时发生 |
| 5 | 事后读 `decisions.md` 的 Overruled 行 | 随意 | 校准对主编的信任（不阻塞管线） |
| **不值** | 大纲细节、文风细部、修订清单逐条把关 | — | 主编已被实测证明干得更快更好：驳回过评审的错误"更正"、否决过委员会散文、正确执行过反煽情红线 |

## 5. 选管线决策树

- **< 3万字** → Autopilot 短篇。
- **3–5万字（灰区）** → 看结构：有"卷级问题"（多个自含弧、各有转折与出口）→
  LONGFORM 缩微试点（《一比一》先例：章/卷规格按比例缩，**门控阈值与修订上限
  永不缩**）；单弧 → 短篇管线。
- **> 5万字** → LONGFORM（规格强制）。
- **目标是网文爆款** → 先花约 1/20 的成本，用一个短篇验证核心设定的新鲜度
  （Originality 能不能上 8），再开 LONGFORM 网文轨 —— 天花板验证要在最便宜的
  尺度上做。

## 6. 规格修订建议汇总（本文只建议；采纳需另行 PR 修改权威文件）

1. **盲评协议加"冻结规则"**：评审包分发前冻结全部修订（出处：《三缄》第 2 轮整轮作废）。
2. **短篇"只败总分 ≥72%"主编验收阀门** —— 或明文声明短/长不对称是有意设计
   （出处：《三缄》45/60=75%，六维全≥7 仍须升级；长篇同分可验收）。
3. **升级呈报模板化**：必须含三选项（定稿/打磨/**重构**）+ 诊断自证（附"去掉最低分
   评审后的中位数重算"——《三缄》的呈报把共识性 45 分误诊为单个苛评员拖底）。
4. **平坦曲线提前升级**：连续两轮只败总分且 Δ总分 ≤1 → 跳过剩余修订轮直接升级。
5. **"2 评审快评"正式化**：人已确认内核的短篇可用 R2+R3 双评审；分歧按低分记，触线升级。
6. **长篇卷 2+ 的 3→2 评审降档条件**：连续两卷 PASS r1 且 R1/R3 中位差 ≤1；
   文学轨限定，网文轨永远保留 R1。

---

**一句话总结**：把人放在天花板上（brief、内核、卷一），把机器放在下限上（纪律、
机械审计、盲评），把预算放在分歧上（评审人次随风险调节，修订及早止损）。
