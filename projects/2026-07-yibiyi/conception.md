# Conception — 《一比一》(Stage L1)

主上下文中 Core 6 + 构思期写作代理（WorldBuilder / PlotArchitect / SymbolWeaver /
CharacterPsychologist / SerialArchitect）就方向的讨论记录。EditorInChief 据此在 decisions.md
锁定内核。

## Diverger（发散：可能性空间）

"数字孪生"是个被用滥的支点，但有一条没被写过的裂缝：**复刻越完美，越暴露它复刻不了的东西。**
往几个方向摊开——
- 横向：谁在被 1:1 扫描？一座即将被水库淹没的村子。100天倒计时是天然的引擎。
- 纵向：扫描仪测得几何、材质、光照，测不到"谁埋在哪、谁没有坟"。于是村里有**另一套记录**：
  一个守碑老人手抄的《名册》，记着谷里所有的死者——包括几十年前上一座坝放水时淹死、至今无坟的人。
  两种"档案"对峙：点云 vs 名册。
- 跨域：测绘 = 会计的空间版。主角信"测得准就是留住了"，正如审计信凭证。这是可继承的声音。
- 逆向：如果主角与这座谷子有一笔**没记进任何册子的私账**呢——她祖父是上一座坝放水那夜的摆渡人。

我倾向：主角是数字化测绘员，被派来把村子做成完美的 1:1 副本；她带着一桩自己都不肯承认的家族旧账
悄悄接了这活。宝藏在"1:1 这个承诺的谎"里。

## Literature Reviewer（证据地基）

三峡/新安江移民、迁坟、"淹没线"红漆是坚实的现实纹理。数字孪生（point cloud / 激光扫描 /
photogrammetry）是成熟技术，其"空心"（点云无内部，手可穿过）是可用的真实意象。上一座 1970s
小坝先淹一次、再被大坝淹第二次——中国不少库区确有"二次淹没"，可信。

## Yes And（连接）

把"会计式主角"接到"守碑人名册"上：两人都是记账的人，记的账不同。主角信可导出、可校验的记录；
老人信一笔一画写下名字这件事本身。让祖父留下一件器物——渡船的铜号牌"浔渡贰号"——把家族旧账物化，
它能在结尾完成一次"迟到几十年的摆渡"。这与 slow-room 的"签收"母题同源而不重样：那里是签收父亲的独白，
这里是签收一册名字。

## Logician（结构校验）

风险：主题会不会沦为"科技冷冰冰、人心才温暖"的口号？必须让主角的"测量信仰"不是愚蠢而是**有尊严的**——
她因此救过东西、也因此看不见东西。让点云真的保住了某些东西（房梁的榫、门的角度），只是保不住"里面"。
对立读法都要成立：数字南屿既是傲慢，也是唯一能带走的东西。

## Challenger（压力测试）

- "祖父是那夜摆渡人"是不是太巧？→ 用**她自己悄悄接活**来承接：不是命运撞上门，是她揣着号牌找上门。
- 守碑人随谷沉没会滑向煽情。→ 不让她殉谷。让她在最后一天离开，把《名册》**签收**给主角——
  由主角把那个无坟者的名字带过"水"，完成祖父没完成的一渡。克制，且把重量交给动作而非眼泪。
- 5万字/2卷能不能真的跑出"卷级"结构？→ 能，只要每卷有独立的卷级问题与出口状态（见 master-outline）。

## Synthesizer（收束，供主编裁决）

共识：一个"1:1 完美副本"的近未来写实中篇，双卷——
- 卷一《测绘》：主角来把村子扫成完美副本，撞见另一套记录（名册）、上一座坝的无坟死者、以及自己家的旧账。
  出口：她认出了那个淹死的名字与那一夜，决定不轮岗撤离，留到沉没。
- 卷二《没顶》：倒计时、迁坟、交底、数字南屿"验收合格"的空心胜利、最后一渡、水漫、以及一件扫描永远
  装不进去的东西被一个人记下来。
分歧（留给文本承担，不升级）：守碑人的去留如何不煽情——方案见 Challenger。
情感目标：**保住了形、丢了实**的钝痛，与一笔迟到几十年的账终于被"人"而非"机器"接住的微光。
