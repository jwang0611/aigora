# LONGFORM 管线测试复盘（总结评论）

用 AIgora Longform 全自动管线，从零跑通一部 **~4.6万字 / 2卷12章** 的文学向迷你长篇《一比一》，
以最小规模走完 **L0 简报 → L1 构思 → L2 卷循环（起草·审计·卷级盲评·压缩）×2 → L3 书级终审 → L4 导出**
全流程。本文记录这次流程测试暴露出来的东西——管线抓住了什么、没抓住什么，以及对 aigora 文档/规格的具体建议。

## 一、这次到底跑了什么

| 阶段 | 实际动作 | 产出 |
|------|----------|------|
| L0/L1 | 主编自简报（文学向、~5万字/2卷）→ Core6+构思代理讨论 → 锁内核/卷地图/6条 PROMISE | brief / conception / story-bible / master-outline / memory 全套 |
| L2 卷一 | 主编亲撰 ch01 立声腔 → **5个并行子代理**起草 ch02-06 → V.AUDIT → **3个盲评子代理**(L1) → 修订 → 压缩 | 6章21.3k字，L1 **65/80** PASS，digest+style-anchor |
| L2 卷二 | **6个并行子代理**起草 ch07-12（加载 style-anchor）→ V.AUDIT → 3盲评 → **1个定向修订子代理** → 压缩 | 6章24.9k字，L1 **68/80** PASS |
| L3/L4 | **3个书级盲评子代理**(B1-B6+抽样漂移) → 机械核验 → 导出 | L2 **50/60** PASS，final/一比一.md |

累计约 **28 个子代理**（12 起草 + 9 卷级盲评 + 3 书级盲评 + 1 定向修订 + 若干），外加主编（主上下文）全程编排、裁决、亲撰 ch01 与全部连续性修订。

## 二、管线真正"抓到了"的东西（有效性证据）

这些是一次性 prompt（one-shot）几乎不可能做到、而流程结构实打实兑现的：

1. **新上下文盲评抓到了主上下文审计看不见的硬伤。** 卷一我在主上下文做 V.AUDIT，只抓到"淹水年份 1972/1970 两种写法"，改成了 1976。**卷一 R3（全新上下文、只读正文）反而抓出我漏的坡顶高程矛盾（313–324 vs 淹没线 326）、迁坟时间线冲突。** 卷二 R3 又抓出 5 条并行起草造成的接缝（水位非单调 322→302、钟阿婆离村的车型/时辰在两章打架、老浔口桩高程差 26 米）。**"盲"确实让评审看见了作者已经看不见的东西**——这是整套设计最硬的一条兑现。

2. **style-anchor + R2 漂移检测，精确命中了它立誓要杀的病。** 卷一过门后，主编从过门文本提炼 style-anchor，并**具名**本书三条易犯 AI 腔（段末对称金句 / 主题被多张嘴复述 / 意象回收）。卷二起草时我把 anchor 喂给了起草代理，仍然犯了 #2 和 #3；**R2 拿 anchor 做漂移比对，一句一句把它们揪了出来**（"补面=名册空条"被叙述者复述两遍、"水无声地涨"意象跨四章）。防线按设计生效了。

3. **多镜片独立收敛＝强信号。** 卷一 R1（素读者）和 R2（工匠）在互不通气的情况下**都**指出"主题过度言明"。三个独立视角撞到同一点，比任何单一评审的自信断言都可信——这正是"三盲取中位"要的东西。

4. **burn-rate / 规模是硬约束，被落到了实处。** 卷一起草完 21.3k < 目标 25k，R≈0.85 被记录并驱动卷二上调章均补足，而不是靠注水——最终 46k 落在"5万字左右"。

5. **封卷 + 抗劣化纪律，挡住了"回炉打磨"。** 书级 R2 对卷一章节提了品味建议（删"互相印证"的自我解经）。按规格，卷一已封（v01-compacted），**不为追味重开冷卷**——这条纪律阻止了长篇里最常见的劣化（越改越顺、越顺越没声音）。规格在这里帮了大忙。

6. **promise-registry 驱动了 100% 伏笔闭合。** 6 条主线伏笔逐卷 plant/escalate/pay 有账可查；"江"姓这条 red herring 被**正式点断**（是别家的江，非主角亲族）而非默默烂尾。

## 三、管线"没抓到 / 反而制造"的东西（诚实的一面）

1. **并行起草打破了"账本即记忆"的地基，制造了真实接缝。** 规格的章循环是**串行**的：起草第 N 章要加载第 N-1 章全文。我为了速度改成**整卷并行**（一次 5–6 个子代理），直接违反了这条——后果就是上面第 II.1 里那一堆接缝：ch07 结尾把主角脚陷在泥里当悬念，ch08 却翻篇当没事；两章各自把水位涨到不同高度；钟阿婆怎么离村，两章描述打架。**盲评当了安全网兜住了，但这些本可以不发生。** 教训：并行起草 ≠ 免费加速，它把"连续性由账本保证"降级成"连续性由事后盲评抢救"。规格应当**更响地警告**这一点，或提供一个专门的"接缝审计"步骤。

2. **对封卷正文的连续性修订，没有自动波及 reader-recap。** 我把卷一日期从"七月十五"改成"八月十四"，改了正文，却**忘了同步改 reader-recap**。结果卷二盲评拿到的前情提要里还是旧的"七月十五"，评审据此报了个"矛盾"。规格的 Retro-Edit 三联更新（账本 Amendments / digest / entity）**没把 reader-recap 列进去**——这是个真实的规格漏洞：reader-recap 是喂给所有下游盲评的，它一旦不同步，会污染之后每一次评审。**建议把 reader-recap 加入 ripple 集。**

3. **"任何评审的连续性发现＝无条件 Must Fix"这条，如果照单全收会引入 bug。** 两个不同的 R3 子代理，对**同一个农历日期**给出了**相反**的公历换算（一个说八月十四对、一个说七月十五对）。真值是八月十四（以唐山地震 1976-07-28＝农历六月初二为锚可定）。**如果我机械执行卷二 R3 的"Must Fix"，就会把正确的改成错误的。** 这说明：连续性 Must Fix 仍然**需要主编对着 ground truth 裁决，不能 auto-apply**。规格把 EditorInChief 说成"在检查点代作者作答"，但这次它真正不可替代的作用，恰恰是在评审彼此矛盾、评审本身出错时**当那个拍板对错的人**。这一点值得在 editor-in-chief.md 里写重。

4. **管线优化的方向，把书推向了"过度对称 / 打磨过头"。** 书级 R2 最犀利的一句不是"哪里塌了"，而是"**它不像没写完，像打磨过头**"——每个符号都精确付清（号牌→渡、木桩→坐标、签收→签收人、梁洞→补面谎言），每格账恰好结平，十二章同一挽歌振幅。量表奖励闭合（B2 伏笔兑现、B6 收束都给到 9），却**几乎没有把力使在"留一道不闭合的口子""让情感有振幅变化"上**。换句话说，**管线会把作品优化成一个鲁棒但单调的高分体**——这可能是 rubric-driven 生成的系统性副作用。建议：量表或主编 mandate 里加一条反"过度确定"的信号（例如允许/鼓励一处 productive incompleteness、一处主角破 register 的失控）。

5. **迷你规模验证了控制流，但验证不了"规模"本身的承诺。** 2卷 46k 跑通了：门控阶梯、盲评、压缩、封卷、导出、伏笔闭合。但它**没有**压到分层记忆的真正痛点——T2/T3 冷加载、digest 膨胀、跨 12 卷的伏笔衰减、compaction 的加载预算硬顶。LONGFORM.md 的 token 预算论证（章 312 与章 12 同价）在这个规模上**没被证伪也没被证实**。

## 四、成本感受

- 一次迷你长篇 ≈ 28 个重型子代理 + 主上下文全程编排。**卷级+书级盲评把起草成本大致翻倍**（3盲评×2卷 + 3书级 = 9 次全卷/全书通读级的评审）。
- 对真正的百万字长篇，这个倍率会随卷数线性放大（14卷×3盲评 + 多轮修订）。LONGFORM.md 自己估的 30–40M tokens/书是可信的量级——这次迷你跑的体感支持"盲评是主要开销之一，绝不能 per-chapter 盲评"这条设计判断。

## 五、给 aigora 的具体改进建议

1. **AUTOPILOT.md / LONGFORM.md 增一节"并行起草的代价"**：明确并行会破坏账本记忆保证，要么保持串行、要么在 V.AUDIT 前加一步显式"接缝审计"（相邻章 entry/exit 对齐 + 同一事件的物理量一致性）。
2. **Retro-Edit 三联更新 → 四联**：把 `memory/reader-recap.md` 加入必须同步的 ripple 集（它污染所有下游盲评）。
3. **editor-in-chief.md 写重"矛盾裁决"职责**：连续性 Must Fix 不 auto-apply；当多个盲评彼此冲突、或涉及可考证事实时，主编须对 ground truth 裁决（本次农历换算即一例）。
4. **rubric 增一条反"过度确定"信号**（或写进主编 Scale Doctrine）：奖励恰当的不闭合与情感振幅，抵消 rubric-driven 生成的"打磨过头"倾向。
5. **模板补一个 `seam-audit` 检查项** 到 longform-quality-gates.md 的 L0，专盯相邻章的物理量/时辰/道具一致性（水位、倒计时、谁带了什么走）。

## 六、结论

**管线兑现了它最核心的承诺：把质量控制搬到"修复还便宜"的地方，并用盲评看见作者看不见的东西。** 连续性硬伤、声纹漂移、伏笔闭合、封卷纪律——都真实生效，且效果可复现。

**但它同样清楚地暴露了两件事：**（1）**"全自动"是有水分的**——真正不可替代的是主编那个**对错裁决 + 对着真值把关**的人类式判断（评审会互相矛盾、会自信地报错）；（2）**rubric-driven 会把作品推向过度对称的高分体**，最像"机器写的"的地方不是哪句烂，而是"什么都押韵、什么都闭合"这件事本身。

一次成功的流程测试，不该只证明它能出活，还该暴露它的系统性偏差。这次两样都做到了：出了一部盲评一致认为"收束配得上前提"的书，也照出了这套管线自己的手癖。
