# AIgora：多智能体智识对话系统

AIgora 是一个面向 Claude 的结构化多智能体讨论框架，将单模型对话转化为多视角、有深度的智识对话。

## 为什么需要 AIgora？

大语言模型擅长生成流畅文本，但在智识深度上存在局限：
- 过快收敛到"合理"答案
- 回避视角之间的生产性张力
- 忽视自身推理的盲区

AIgora 通过编排**多个具有不同认知角色的智能体**，创建结构化对话——让观点被探索、挑战、整合，而非仅仅被生成。

## 核心架构

### 固定发言顺序

AIgora 强制执行一个刻意设计的发言顺序，模拟有效智识对话的自然展开：

```
用户输入
    ↓
[1] 发散者 ────────── 打开可能性空间（始终第一）
    ↓
[弹性区域] ─────── 文献评审员、Yes And、逻辑学家、专家
    ↓
[N-1] 挑战者 ────── 压力测试一切（始终倒数第二）
    ↓
[N] 综合者 ──────── 关闭回合（始终最后）
```

**为何这个顺序重要：**
- **发散者在前**：防止过早收敛。讨论必须先打开再收窄。
- **挑战者倒数第二**：所有材料必须上桌后才进行压力测试。
- **综合者最后**：综合之后本回合不再有发言——它为下一轮迭代做准备。

### 核心六智能体

| 智能体 | 认知角色 | 关键行为 |
|-------|---------|---------|
| **发散者** | 打开可能性空间 | 探索横向、纵向、跨领域、逆向方向。在映射地形后表明倾向。 |
| **文献评审员** | 提供证据基础 | 将讨论锚定于已有知识、研究和先例。 |
| **Yes And** | 建设与连接 | 寻找生产性组合。建设性地延展观点。 |
| **逻辑学家** | 分析论证结构 | 审视有效性，识别谬误，澄清推理链。 |
| **挑战者** | 压力测试一切 | 质疑假设，压力测试论证，提供替代方案。 |
| **综合者** | 关闭回合 | 提取核心主张，识别共识/分歧，准备下一步。 |

### 三种专业模式

**通用讨论模式**（默认）
- 核心6 + 话题相关专家
- 适用：哲学、政策分析、复杂决策

**小说写作模式**（触发词：小说、story、character、scene...）
- 核心6 + 10个写作智能体 + 文学专家
- 强制：AIDetox、VoiceDistinctor、KnowledgeAuditor、FirstReader、CharacterPsychologist、EmotionMaster
- 阶段性：构思 / 写作 / 修订 智能体

**游戏设计模式**（触发词：游戏设计、game mechanics、balance...）
- 核心3（发散者、挑战者、综合者）+ 6个游戏设计智能体
- Visionary → MechanicSmith → Economist → Psychologist → Breaker → Producer

### 全自动写作模式（Autopilot）

**触发词**：自动写作、全自动小说、autopilot

小说流水线的全自动版本：人类只写一份简报（甚至一句"你来定"），流水线端到端运行，
人工只在必要时介入。详见 `AUTOPILOT.md`。

- **主编（EditorInChief）**代替人类回答守门人（Gatekeeper）的所有检查点决策
  （故事内核、情感目标、结局、结构、POV、修订取舍），每个决策连同理由记入 `decisions.md`
- **量化质量门**（`references/core/quality-rubric.md`）：结构 / 人物 / 文笔 / 情感冲击 /
  原创性去AI味 / 主题深度 六维各10分；总分 ≥ 48/60 且单项 ≥ 7 才算通过；修订循环上限 3 轮
- **并行盲评**：评审由 3 个全新上下文的独立评审员完成（Claude Code 中为并行子代理），
  取各维度中位数——写作现场的评分是被污染的，盲评才可信
- **落盘记忆**：每部作品一个 `projects/<slug>/` 目录（故事圣经、大纲、连续性账本、逐章文件、
  评审记录、终稿），长篇不再受上下文窗口限制
- **人工介入点仅四处**：3 轮修订仍不达标 / 涉及作者本人价值判断的题材 / 简报中标注
  "必须问我"的决策 / 简报本身

## 智能体哲学

### 智能体有立场

与中性的"视角"不同，AIgora 智能体是**有观点的**：
- 发散者表明它认为宝藏在哪里
- 挑战者言出必行——不是扮演魔鬼代言人
- 专家智能体体现真实的智识立场

### 智能体互动

鼓励直接对话：
- 挑战者可能反对发散者的倾向
- 专家可以建设或批评彼此
- 综合者评估论证质量（而非仅仅总结）

### 约束产生深度

固定顺序和角色边界创造生产性约束：
- 发散者不能批评（那是挑战者的工作）
- 综合者不能添加新想法（只能处理桌上的材料）
- 每个智能体深入做一件事，而非浅尝辄止地做所有事

## 82个专家智能体

除核心六智能体外，AIgora 包含按领域组织的专家智能体：

| 类别 | 数量 | 示例 |
|------|-----|------|
| **思想家** | 17 | 马克思、哈耶克、阿伦特、福柯、韦伯、波普尔... |
| **哲学家** | 22 | 柏拉图、亚里士多德、康德、海德格尔、维特根斯坦... |
| **作家** | 26 | 荷马、陀思妥耶夫斯基、博尔赫斯、刘慈欣、金庸... |
| **研究者** | 17 | 计量经济学家、博弈论专家、未来学家... |

专家根据话题相关性被选择，在弹性区域发言。

## 设计原则

**1. 结构优先于自由**
刚性发言顺序为真正的探索创造空间。智能体无法做所有事，所以深入做好自己的角色。

**2. 张力优先于共识**
挑战者的存在是为了防止过早达成一致。分歧被映射，而非被消解。

**3. 实质优先于表演**
智能体不是戏剧性地扮演角色——它们体现不同的认知功能。

**4. 迭代优先于完成**
每一轮以"下一步"结束。综合者准备场地；用户决定去向。

## 定制

你可以：
- 在 `references/` 中添加自定义专家智能体
- 修改智能体定义以适应你的需求
- 创建新模式（如：法律分析、学术评审）
